# ve-forwardrole

## What this is
A quality-first, **multi-profile** job-search assistant. Run it for yourself and
your friends without mixing anyone's data. Not a mass-apply bot — one
well-tailored application beats ten generic ones.

New here? Read **README.md** for the two-tier quickstart and
**docs/HOW-IT-WORKS.md** for the architecture and why each tool is in the stack.

## Two ways to run it
- **Tier 1 — Claude Code only** (no accounts, no server): drop a LinkedIn PDF,
  run the agents in `.claude/agents/`, tailor applications by hand-picked URL.
  This is the flow the "Workflow" sections below describe.
- **Tier 2 — the autonomous pipeline** (`pi/`) + dashboard (`app/`): a cron
  discovers roles from 4 sources, triages them, maps warm intros, scores fit,
  and serves a per-person dashboard. Each capability is unlocked by one optional
  key — see `.env.example`. Nothing is required; absence of a key just disables
  that one capability.

## Tech stack
- TypeScript (strict) + ts-node for the pipeline (`pi/`) and coordinator (`src/`)
- Claude Code agents (`.claude/agents/`) for parse/read/tailor/write/track
- Next.js dashboard (`app/`) + PocketBase auth (`pb_migrations/`)
- LinkedIn PDF input (no scraping for the CV); optional Apify/Exa for discovery
- LLM via any one of ANTHROPIC_API_KEY / OPENAI_API_KEY / LLM_BASE_URL

## File layout
```
profiles/<name>/        one person's data (gitignored except example/)
  linkedin.pdf          input — LinkedIn "Save to PDF" export
  profile.md            parsed CV (generated by linkedin-parser)
  search.yaml           what to look for (see profiles/example/search.example.yaml)
  applications.md       outcome log
  output/               tailored CVs + cover letters
jobs/queue.md           discovered/added roles, tagged "| profile=<name>"
.claude/agents/         the five Claude Code subagents
src/                    queue validation (npm run check)
pi/                     the autonomous pipeline (discover → triage → enrich → render)
app/                    the Next.js dashboard
docs/HOW-IT-WORKS.md    architecture + tool rationale
```

## Workflow (per profile, one-time setup, Tier 1)
1. `mkdir -p profiles/<name>` and drop their `linkedin.pdf` in it
   (LinkedIn → More → Save to PDF). Copy `profiles/example/search.example.yaml`
   to `profiles/<name>/search.yaml` and edit it.
2. In Claude Code: `Run linkedin-parser on profiles/<name>/linkedin.pdf`
3. Inspect the generated `profiles/<name>/profile.md`, hand-edit anything off.

## Workflow (per job, Tier 1)
1. Add the URL to `jobs/queue.md` tagged `| profile=<name>`
2. `Run job-reader on <url>` → job JSON
3. `Run cv-tailor for profile=<name> with <job JSON>` → tailored CV + reasoning + match score
4. `Run cover-letter-writer for profile=<name>` → cover letter
5. `Run application-tracker for profile=<name>` → logs to `profiles/<name>/applications.md`

## Agents
| Agent | File | Purpose |
|-------|------|---------|
| linkedin-parser | `.claude/agents/linkedin-parser.md` | LinkedIn PDF → structured `profile.md` |
| job-reader | `.claude/agents/job-reader.md` | Job URL → structured job JSON |
| cv-tailor | `.claude/agents/cv-tailor.md` | Reorders/reframes `profile.md` for the role |
| cover-letter-writer | `.claude/agents/cover-letter-writer.md` | Writes a 3-paragraph cover letter |
| application-tracker | `.claude/agents/application-tracker.md` | Appends to per-profile `applications.md` |

## Hard rules
- `linkedin-parser` and `cv-tailor` MUST NEVER invent skills, jobs, or dates
- Always show tailoring reasoning before the tailored CV
- Always log to the right profile's `applications.md` — never mix profiles
- Match score below 5 = flag and skip unless deliberate
- Friends' data lives only under `profiles/<name>/` and is gitignored
- Keep your copy of this repo **private** — the queue holds a real job hunt
