# ve-forwardrole

## What this is
A Claude Code native skill for quality-first job applications, **multi-profile**.
Built to run applications for multiple people (you + friends) without mixing their data.
Not a mass-apply bot. One well-tailored application beats ten generic ones.

## Tech stack
- TypeScript (strict) + ts-node
- Claude Code agents (`.claude/agents/`)
- LinkedIn PDF input (no scraping, no LinkedIn API)
- No external APIs — runs entirely through Claude Code

## File layout
```
profiles/
  <name>/
    linkedin.pdf       ← input (LinkedIn "Save to PDF" export)
    profile.md         ← parsed CV (generated by linkedin-parser)
    applications.md    ← outcome log for this person
    output/            ← tailored CVs + cover letters for this person
jobs/
  queue.md             ← URLs tagged with "| profile=<name>"
.claude/agents/        ← agent prompts
src/                   ← TypeScript coordinator
```

## Dev commands
```bash
npm install
npm run check:dry                 # preview entire queue
npm run check                     # validate every profile + queue, print instructions
npm run check -- --profile=ade    # only this profile
```

## Workflow (per profile, one-time setup)
1. `mkdir profiles/<name>`
2. Save LinkedIn → "Save to PDF" → drop as `profiles/<name>/linkedin.pdf`
3. In Claude Code: `Run linkedin-parser on profiles/<name>/linkedin.pdf`
4. Inspect generated `profiles/<name>/profile.md`, hand-edit anything off

## Workflow (per job)
1. Add the URL to `jobs/queue.md` tagged `| profile=<name>`
2. `Run job-reader on <url>` → job JSON
3. `Run cv-tailor for profile=<name> with <job JSON>` → tailored CV + reasoning + match score
4. `Run cover-letter-writer for profile=<name>` → cover letter
5. `Run application-tracker for profile=<name>` → logs to `profiles/<name>/applications.md`

## Agents
| Agent | File | Purpose |
|-------|------|---------|
| linkedin-parser | `.claude/agents/linkedin-parser.md` | LinkedIn PDF → structured `profile.md` |
| job-reader | `.claude/agents/job-reader.md` | Job URL → structured job JSON |
| cv-tailor | `.claude/agents/cv-tailor.md` | Reorders/reframes `profile.md` for the role |
| cover-letter-writer | `.claude/agents/cover-letter-writer.md` | Writes a 3-paragraph cover letter |
| application-tracker | `.claude/agents/application-tracker.md` | Appends to per-profile `applications.md` |

## Hard rules
- `linkedin-parser` and `cv-tailor` MUST NEVER invent skills, jobs, or dates
- Always show tailoring reasoning before the tailored CV
- Always log to the right profile's `applications.md` — never mix profiles
- Match score below 5 = flag and skip unless deliberate
- Friends' data lives only under `profiles/<name>/` and is gitignored
