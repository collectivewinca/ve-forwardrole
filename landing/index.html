The 07:00 run finds 40 roles across three sources. Triage kills 25 — wrong seniority, below your floor, no sponsorship. The 15 survivors get fit scores and warm paths, your dashboard re-ranks, and your morning Telegram reads: "1. Product Lead @ Canary · fit 9/10 · 5 warm paths." You star two, skip six (skips are remembered — they never come back), send a drafted note to the ex-colleague at your top pick, and tailor one application in Claude Code. Ten minutes. All signal.

the stack, justified

Why these tools — and not alternatives

Every dependency is optional and earns its place. Remove a key and you lose a capability, never the system.

Claude / Claude Code

the judgment

The hard work here is judgment, not retrieval — "is Director-level right for this person?", "reframe without inventing". The bundled agents enforce rules a template engine can't: never fabricate experience, always show reasoning. And the repo is the interface — open it in Claude Code and the agents just load.

ATS public boardsfree

the default firehose

Greenhouse, Lever, Ashby and SmartRecruiters expose their boards as public JSON by design. The only unlimited, ToS-clean, zero-account source — one watchlist company = its entire board, every run. This is the default source precisely because it costs nothing.

Apifyoptional

linkedin discovery

LinkedIn has no public jobs API and aggressively blocks scrapers. Apify maintains the actors and rotates the infrastructure — you rent the cat-and-mouse game instead of playing it. Drop it and the other two sources still feed you.

Exafree tier

the network engine

Keyword search can't answer "people who worked at Mastercard, now at Pagaya" or "Cornell alumni at Datadog in senior product roles." Exa's neural search can — with per-result facts the pipeline verifies before showing you a name. This is what turns a job list into a network strategy.

Any LLM providerbyom

no vendor lock

The brain resolves whichever you set: an OpenAI-compatible gateway (OpenRouter, Groq, a local Ollama — free), an Anthropic key, or an OpenAI key. One env variable, your choice of model economics.

PocketBasedashboard only

auth in one binary

Login, admin UI and migrations in a single executable. The alternative is running Postgres plus an auth service for what amounts to five user rows per instance. ./pocketbase serve — done.

git + pull requests

the audit trail

Every discovery run is a commit; every queue change is a diff. The robot's decisions are reviewable and revertable — a database would hide them, a markdown file in git makes the robot accountable by reading the history.

cron, Telegram, here.nowoptional

the plumbing

Boring on purpose. cron because the pipeline is idempotent scripts; Telegram because a morning push shouldn't require building an app; here.now so a mentor can see your shortlist behind a password without getting a dashboard login.

design principles

Why you can trust it with your job hunt

Degrades gracefullyThe free tier — ATS boards + Claude Code — is a complete product. Keys add capabilities; their absence never breaks the system.
Never inventsThe parser and tailor are constitutionally forbidden from fabricating skills, dates, or jobs. Tailoring reorders and reframes what's true.
Files over databasesProfiles, queue, decisions — markdown and JSON in your repo. Greppable, diffable, portable, yours.
Bring your own modelAnthropic, OpenAI, OpenRouter, or a local Ollama. One variable; never locked to a vendor.
Private by defaultOne instance = one operator + their people. No SaaS, no telemetry, no shared backend.
Quality over volumeIt will never mass-apply. The constraint is the feature: ten minutes a day, all signal.

get started

Ten minutes, zero new accounts

  1. On GitHub, click "Use this template" → create a private copy → clone it.
  2. LinkedIn → More → Save to PDF → drop it at profiles/me/linkedin.pdf.
  3. Open the repo in Claude Code and say: Run linkedin-parser on profiles/me/linkedin.pdf
  4. Paste a job URL into jobs/queue.md and ask for a tailored CV. That's the whole loop.
  5. Later, if you want the robot: bash setup.sh, add any one LLM key, set the cron. The README walks you through it.

full setup reference

Every command you'll need

Copy-paste friendly. The repo: github.com/collectivewinca/ve-forwardrole

Create your private copy (CLI route)

gh repo create YOU/my-forwardrole --private \
  --template collectivewinca/ve-forwardrole --clone
cd my-forwardrole && npm install

Make it conversational — install the bundled Claude Code skill

cp -r skill/forwardrole ~/.claude/skills/

Then, from any directory: "tailor my CV for <url>", "who do I know at Datadog?", "find me roles" — Claude Code drives the whole workflow.

Turn on the robot (any Linux box or VPS)

bash setup.sh                      # deps + .env + capability checklist
# .env — set ANY ONE brain:
#   ANTHROPIC_API_KEY=...            (recommended)
#   OPENAI_API_KEY=...
#   LLM_BASE_URL=http://localhost:11434/v1  LLM_MODEL=llama3.1   (free, local Ollama)

bash pi/onboard.sh --name=me --pdf=profiles/me/linkedin.pdf \
  --keywords="head of product, VP product" --locations="New York, Remote"

crontab -e                          # twice-daily; keep commands in the script —
0 7,19 * * * bash $HOME/my-forwardrole/pi/run-pipeline.sh >> $HOME/my-forwardrole/pi/cron.log 2>&1
